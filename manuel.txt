SECTION 1 — PRÉSENTATION DU BACKEND
================================================================================

1.1 CONTEXTE ET OBJECTIF
--------------------------
Teranga GESCRIM est un système d'information complet destiné à la Direction de
la Sécurité Publique (DSP) du Sénégal. Il permet aux agents de police et de
sécurité de saisir, gérer et analyser les données relatives à :
  - Les infractions constatées ou déférées
  - Les accidents de la circulation
  - L'immigration clandestine
  - Le personnel des services de sécurité
  - Les amendes et pièces saisies
  - Les services rémunérés

L'application supporte le mode hors-ligne (offline) grâce à un système de
synchronisation par lot (batch sync), ce qui est essentiel pour les agents
terrain qui n'ont pas toujours accès à Internet.

1.2 ARCHITECTURE GÉNÉRALE
---------------------------
Le backend est une API REST stateless (sans état de session) construite avec
le framework PHP Laravel 13. Les principales caractéristiques architecturales :

  - Architecture : MVC (Modèle-Vue-Contrôleur) adapté en API-REST
  - Authentification : JWT (JSON Web Tokens) — stateless, sécurisé
  - Base de données : PostgreSQL avec Eloquent ORM
  - Rôles et permissions : Spatie Permission (RBAC)
  - Documentation : Swagger/OpenAPI via L5-Swagger
  - Export : PDF via DomPDF, CSV natif PHP
  - Synchronisation offline : Endpoint batch pour les applications mobiles

1.3 TECHNOLOGIES UTILISÉES
----------------------------
  Langage         : PHP 8.3
  Framework       : Laravel 13.x
  Base de données : PostgreSQL 14+
  Authentification: php-open-source-saver/jwt-auth
  Rôles           : spatie/laravel-permission
  Documentation   : darkaonline/l5-swagger + zircote/swagger-php
  Export PDF      : barryvdh/laravel-dompdf
  Annotations     : doctrine/annotations
  Tests           : PHPUnit 12.x
  Gestionnaire dep: Composer 2.x
  Serveur dev     : php artisan serve (port 8000)

1.4 FONCTIONNEMENT GLOBAL
---------------------------
1. Un client (Angular ou Flutter) envoie une requête HTTP à l'API.
2. Laravel reçoit la requête via le fichier d'entrée public/index.php.
3. Le router (routes/api.php) dirige la requête vers le contrôleur adéquat.
4. Le middleware auth:api vérifie le token JWT dans l'en-tête Authorization.
5. Le contrôleur valide les données, exécute la logique métier via Eloquent.
6. Eloquent ORM effectue les requêtes SQL vers PostgreSQL.
7. Le trait Auditable enregistre automatiquement toute modification dans audit_logs.
8. Le contrôleur retourne une réponse JSON standardisée.

Format standard de toutes les réponses :
  Succès : { "success": true, "message": "...", "data": {...} }
  Erreur : { "success": false, "message": "...", "errors": {...} }
  Paginé : { "success": true, "data": [...], "meta": { pagination... } }

SECTION 4 — INSTALLATION DES DÉPENDANCES
================================================================================

4.1 JWT AUTH — Authentification par Token
------------------------------------------
composer require php-open-source-saver/jwt-auth

Rôle : Permet d'authentifier les utilisateurs sans session serveur.
Chaque utilisateur reçoit un token JWT signé lors de la connexion.
Ce token est inclus dans chaque requête dans le header Authorization.

Après installation :
  # Publier le fichier de configuration jwt.php
  php artisan vendor:publish --provider="PHPOpenSourceSaver\JWTAuth\Providers\LaravelServiceProvider"

  # Générer la clé secrète JWT (ajoutée dans .env comme JWT_SECRET)
  php artisan jwt:secret

4.2 SPATIE LARAVEL PERMISSION — Rôles et Permissions
------------------------------------------------------
composer require spatie/laravel-permission

Rôle : Système RBAC (Role-Based Access Control) complet.
Permet de définir des rôles (super_admin, admin, superviseur, agent)
et des permissions granulaires (infractions.create, users.delete, etc.).

Après installation :
  # Publier les fichiers de configuration et migrations
  php artisan vendor:publish --provider="Spatie\Permission\PermissionServiceProvider"

4.3 L5-SWAGGER — Documentation API Automatique
------------------------------------------------
composer require darkaonline/l5-swagger

Rôle : Génère automatiquement une interface Swagger UI à partir des
annotations @OA dans les contrôleurs. Accessible sur /api/documentation.

Après installation :
  php artisan vendor:publish --provider="L5Swagger\L5SwaggerServiceProvider"

4.4 LARAVEL DOMPDF — Export PDF
---------------------------------
composer require barryvdh/laravel-dompdf

Rôle : Convertit des vues HTML Blade en fichiers PDF téléchargeables.
Utilisé pour exporter les listes d'infractions et d'accidents.

4.5 DOCTRINE ANNOTATIONS — Requis par Swagger
-----------------------------------------------
composer require doctrine/annotations

Rôle : Bibliothèque PHP pour parser les annotations dans les commentaires
de code (les blocs /** @OA\... */ que l5-swagger utilise).

4.6 DÉPENDANCES DE DÉVELOPPEMENT (déjà incluses)

  phpunit/phpunit   : Exécuter les tests unitaires et fonctionnels

FICHIER : config/auth.php
----------------------------------------------------------------------
Chemin exact : backendmemoire/config/auth.php
Rôle : Configure les guards d'authentification. Ici on change le guard
"api" pour utiliser le driver "jwt" au lieu du driver "token" par défaut.
----------------------------------------------------------------------
<?php
use App\Models\User;
return [
    'defaults' => [
        'guard' => env('AUTH_GUARD', 'api'),
        'passwords' => env('AUTH_PASSWORD_BROKER', 'users'),
    ],
    'guards' => [
        'web' => [
            'driver' => 'session',
            'provider' => 'users',
        ],
        'api' => [
            'driver' => 'jwt',        // <-- Clé : on utilise JWT
            'provider' => 'users',
        ],
    ],
    'providers' => [
        'users' => [
            'driver' => 'eloquent',
            'model' => env('AUTH_MODEL', User::class),
        ],
    ],
    'passwords' => [
        'users' => [
            'provider' => 'users',
            'table' => 'password_reset_tokens',
            'expire' => 60,
            'throttle' => 60,
        ],
    ],
    'password_timeout' => env('AUTH_PASSWORD_TIMEOUT', 10800),
];

Explication :
  'guard' => 'api'       : Par défaut, utiliser le guard "api"
  'driver' => 'jwt'      : Ce guard utilise les tokens JWT (pas les sessions)
  'provider' => 'users'  : Les utilisateurs viennent du modèle User via Eloquent
  Le middleware auth:api  : Vérifiera automatiquement le token JWT dans les headers

----------------------------------------------------------------------
FICHIER : config/cors.php
----------------------------------------------------------------------
Chemin exact : backendmemoire/config/cors.php
Rôle : Configure les autorisations Cross-Origin Resource Sharing.
Permet à l'application Angular et Flutter d'appeler l'API depuis
des domaines différents.
----------------------------------------------------------------------
<?php
return [
    'paths' => ['api/*', 'sanctum/csrf-cookie'],
    'allowed_methods' => ['*'],
    'allowed_origins' => ['*'],
    'allowed_origins_patterns' => [],
    'allowed_headers' => ['*'],
    'exposed_headers' => [],
    'max_age' => 0,
    'supports_credentials' => false,
];

Explication :
  'paths' => ['api/*']       : CORS s'applique uniquement aux routes /api/...
  'allowed_origins' => ['*'] : Tous les domaines sont autorisés (dev)
  'allowed_methods' => ['*'] : GET, POST, PUT, DELETE, OPTIONS autorisés

FICHIER : app/Http/Controllers/Api/ApiController.php
----------------------------------------------------------------------
Chemin exact : backendmemoire/app/Http/Controllers/Api/ApiController.php
Rôle : Contrôleur parent de tous les contrôleurs API. Fournit trois
méthodes utilitaires pour formater les réponses JSON de manière
standardisée dans toute l'application. Contient aussi toutes les
annotations Swagger globales (@OA\Info, @OA\SecurityScheme, schémas).
----------------------------------------------------------------------
<?php
namespace App\Http\Controllers\Api;

use Illuminate\Http\JsonResponse;
use Illuminate\Routing\Controller as BaseController;
use OpenApi\Annotations as OA;

/**
 * @OA\Info(
 *     title="Teranga GESCRIM API",
 *     version="1.0.0",
 *     description="API REST pour la DSP du Sénégal."
 * )
 * @OA\Server(url=L5_SWAGGER_CONST_HOST)
 * @OA\SecurityScheme(
 *     securityScheme="bearerAuth",
 *     type="http",
 *     scheme="bearer",
 *     bearerFormat="JWT"
 * )
 */
class ApiController extends BaseController
{
    // Réponse de succès : HTTP 200 par défaut
    protected function successResponse($data = null, string $message = 'Opération réussie', int $code = 200): JsonResponse
    {
        return response()->json([
            'success' => true,
            'message' => $message,
            'data'    => $data,
        ], $code);
        // Exemple de réponse :
        // { "success": true, "message": "Infraction créée.", "data": {...} }
    }

    // Réponse d'erreur : HTTP 400 par défaut
    protected function errorResponse(string $message = 'Erreur', int $code = 400, $errors = null): JsonResponse
    {
        $response = ['success' => false, 'message' => $message];
        if ($errors) {
            $response['errors'] = $errors;
        }
        return response()->json($response, $code);
        // Exemple : { "success": false, "message": "Validation", "errors": {...} }
    }

    // Réponse paginée avec métadonnées de pagination
    protected function paginatedResponse($paginator, string $message = 'Données récupérées'): JsonResponse
    {
        return response()->json([
            'success' => true,
            'message' => $message,
            'data'    => $paginator->items(),     // Les enregistrements de la page
            'meta'    => [
                'current_page' => $paginator->currentPage(),  // Page actuelle
                'last_page'    => $paginator->lastPage(),      // Dernière page
                'per_page'     => $paginator->perPage(),       // Items par page
                'total'        => $paginator->total(),         // Total d'enregistrements
                'from'         => $paginator->firstItem(),     // Numéro du 1er item
                'to'           => $paginator->lastItem(),      // Numéro du dernier item
            ],
        ]);
    }
}

FICHIER : app/Traits/Auditable.php
----------------------------------------------------------------------
Chemin exact : backendmemoire/app/Traits/Auditable.php
Rôle : Trait PHP réutilisable. Quand on l'applique sur un modèle Eloquent
avec "use Auditable;", ce modèle enregistre AUTOMATIQUEMENT toute action
(création, modification, suppression) dans la table audit_logs.
Aucun code supplémentaire n'est nécessaire dans les contrôleurs.
----------------------------------------------------------------------
<?php
namespace App\Traits;

use App\Models\AuditLog;
use Illuminate\Support\Facades\Auth;
use Illuminate\Support\Facades\Request;

trait Auditable
{
    // Méthode appelée automatiquement au boot du modèle Eloquent
    public static function bootAuditable(): void
    {
        // ÉVÉNEMENT 1 : Enregistrement lors d'une création
        static::created(function ($model) {
            self::logAudit('create', $model, null, $model->getAttributes());
        });

        // ÉVÉNEMENT 2 : Enregistrement lors d'une mise à jour
        static::updated(function ($model) {
            $original = $model->getOriginal();   // Valeurs avant modif
            $changes  = $model->getChanges();    // Valeurs après modif
            if (!empty($changes)) {
                self::logAudit('update', $model, $original, $changes);
            }
        });

        // ÉVÉNEMENT 3 : Enregistrement lors d'une suppression
        static::deleted(function ($model) {
            self::logAudit('delete', $model, $model->getAttributes(), null);
        });
    }

    // Méthode privée qui insère réellement le log
    protected static function logAudit(string $action, $model, ?array $oldValues, ?array $newValues): void
    {
        try {
            AuditLog::create([
                'user_id'    => Auth::id(),              // Qui a fait l'action
                'action'     => $action,                 // create/update/delete
                'model_type' => get_class($model),       // Quel modèle (App\Models\Infraction)
                'model_id'   => $model->getKey(),        // ID de l'enregistrement modifié
                'old_values' => $oldValues,              // État avant (JSON)
                'new_values' => $newValues,              // État après (JSON)
                'ip_address' => Request::ip(),           // Adresse IP de l'auteur
                'user_agent' => Request::userAgent(),    // Navigateur/App utilisée
            ]);
        } catch (\Exception $e) {
            // L'audit ne doit pas bloquer l'opération principale
            \Log::error('Audit log error: ' . $e->getMessage());
        }
    }
}

use Illuminate\Notifications\Notifiable;
use PHPOpenSourceSaver\JWTAuth\Contracts\JWTSubject;
use Spatie\Permission\Traits\HasRoles;
use App\Traits\Auditable;

class User extends Authenticatable implements JWTSubject
{
    use Notifiable, HasRoles, Auditable;

    protected $fillable = [
        'name', 'email', 'password', 'telephone',
        'service_id', 'is_active', 'last_login_at',
    ];

    // Champs exclus des réponses JSON (sécurité)

    {
        return [
            'name'  => $this->name,
            'email' => $this->email,
            'roles' => $this->getRoleNames(),
        ];
    }

    public function service()              { return $this->belongsTo(Service::class); }
    public function personnel()            { return $this->hasOne(Personnel::class); }
    public function notificationsInternes(){ return $this->hasMany(NotificationInterne::class); }
    public function auditLogs()            { return $this->hasMany(AuditLog::class); }

ENDPOINT 1 : POST /api/auth/login
Méthode : POST
URL : http://localhost:8000/api/auth/login
Authentification : Aucune (route publique)
Body JSON :
  { "email": "admin@gescrim.sn", "password": "password123" }
Validation :
  email    : obligatoire, format email valide
  password : obligatoire, minimum 6 caractères
Logique interne :
  1. Valider les données reçues
  2. Vérifier que l'utilisateur existe et est actif (is_active=true)
  3. Tenter l'authentification JWT (JWTAuth::attempt)
  4. Si échec : enregistrer un AuditLog "login_failed" + retourner 401
  5. Si succès : mettre à jour last_login_at + AuditLog "login" + retourner token
Réponse succès (200) :
  {
    "success": true,
    "message": "Connexion réussie.",
    "data": {
      "access_token": "eyJ0eXAiOiJKV1QiLCJhbGci...",
      "token_type": "bearer",
      "expires_in": 3600,
      "user": { "id": 1, "name": "Admin Principal", "email": "...", "roles": ["super_admin"] }
    }
  }
Réponse erreur (401) :
  { "success": false, "message": "Identifiants incorrects." }
Réponse erreur (403) :
  { "success": false, "message": "Votre compte est désactivé." }
Réponse erreur (422) :
  { "success": false, "message": "Erreur de validation", "errors": { "email": [...] } }

ENDPOINT 2 : POST /api/auth/logout
Méthode : POST
URL : http://localhost:8000/api/auth/logout
Header : Authorization: Bearer <token>
Logique : Enregistre un AuditLog "logout", invalide le token JWT.
Réponse (200) : { "success": true, "message": "Déconnexion réussie." }

ENDPOINT 3 : POST /api/auth/refresh
Méthode : POST
URL : http://localhost:8000/api/auth/refresh
Header : Authorization: Bearer <token_expiré>
Logique : Génère un nouveau token sans que l'utilisateur se reconnecte.
Réponse (200) : Même structure que login (nouveau access_token)

ENDPOINT 4 : GET /api/auth/me
Méthode : GET
URL : http://localhost:8000/api/auth/me
Header : Authorization: Bearer <token>
Logique : Retourne le profil complet de l'utilisateur connecté avec
ses rôles, permissions, service et personnel associé.
Réponse (200) :
  {
    "success": true,
    "data": {
      "id": 1, "name": "...", "email": "...", "telephone": "...",
      "is_active": true, "last_login_at": "2026-05-19T...",
      "service": {...}, "roles": ["super_admin"],
      "permissions": ["infractions.view", "users.create", ...],
      "personnel": {...}
    }
  }

ENDPOINT 5 : POST /api/auth/change-password
Méthode : POST
URL : http://localhost:8000/api/auth/change-password
Header : Authorization: Bearer <token>
Body JSON :
  {
    "current_password": "motDePasseActuel",
    "new_password": "nouveauMotDePasse",
    "new_password_confirmation": "nouveauMotDePasse"
  }
Validation :
  current_password           : obligatoire
  new_password               : obligatoire, min 8 caractères, doit être confirmé
  new_password_confirmation  : doit correspondre à new_password
Logique : Vérifie que le mot de passe actuel est correct (Hash::check),
puis hache et enregistre le nouveau (Hash::make).
Réponse (200) : { "success": true, "message": "Mot de passe modifié avec succès." }
Réponse (400) : { "success": false, "message": "Le mot de passe actuel est incorrect." }

CODE COMPLET DE AuthController.php :
<?php
namespace App\Http\Controllers\Api\Auth;

use App\Http\Controllers\Api\ApiController;
use App\Models\AuditLog;
use App\Models\User;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Hash;
use Illuminate\Support\Facades\Validator;
use PHPOpenSourceSaver\JWTAuth\Facades\JWTAuth;

class AuthController extends ApiController
{
    public function __construct()
    {
        // Toutes les méthodes requièrent un token JWT, SAUF login
        $this->middleware('auth:api')->except(['login']);
    }

    public function login(Request $request): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'email'    => 'required|email',
            'password' => 'required|string|min:6',
        ], [
            'email.required'    => "L'email est obligatoire.",
            'email.email'       => "L'email doit être valide.",
            'password.required' => 'Le mot de passe est obligatoire.',
            'password.min'      => 'Le mot de passe doit contenir au moins 6 caractères.',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $credentials = $request->only('email', 'password');

        // Vérifier si le compte est actif AVANT d'essayer le token
        $user = User::where('email', $credentials['email'])->first();
        if ($user && !$user->is_active) {
            return $this->errorResponse("Votre compte est désactivé. Contactez l'administrateur.", 403);
        }

        if (!$token = JWTAuth::attempt($credentials)) {
            AuditLog::create([
                'user_id' => null, 'action' => 'login_failed',
                'model_type' => User::class, 'model_id' => null,
                'new_values' => ['email' => $credentials['email']],
                'ip_address' => $request->ip(), 'user_agent' => $request->userAgent(),
            ]);
            return $this->errorResponse('Identifiants incorrects.', 401);
        }

        $user = auth()->user();
        $user->update(['last_login_at' => now()]);
        AuditLog::create([
            'user_id' => $user->id, 'action' => 'login',
            'model_type' => User::class, 'model_id' => $user->id,
            'ip_address' => $request->ip(), 'user_agent' => $request->userAgent(),
        ]);

        return $this->respondWithToken($token);
    }

    public function logout(Request $request): JsonResponse
    {
        $user = auth()->user();
        AuditLog::create([
            'user_id' => $user->id, 'action' => 'logout',
            'model_type' => User::class, 'model_id' => $user->id,
            'ip_address' => $request->ip(), 'user_agent' => $request->userAgent(),
        ]);
        auth()->logout();
        return $this->successResponse(null, 'Déconnexion réussie.');
    }

    public function refresh(): JsonResponse
    {
        return $this->respondWithToken(auth()->refresh());
    }

    public function me(): JsonResponse
    {
        $user = auth()->user();
        $user->load(['service', 'roles', 'personnel']);
        return $this->successResponse([
            'id' => $user->id, 'name' => $user->name, 'email' => $user->email,
            'telephone' => $user->telephone, 'is_active' => $user->is_active,
            'last_login_at' => $user->last_login_at, 'service' => $user->service,
            'roles' => $user->getRoleNames(),
            'permissions' => $user->getAllPermissions()->pluck('name'),
            'personnel' => $user->personnel,
        ]);
    }

    public function changePassword(Request $request): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'current_password' => 'required|string',
            'new_password'     => 'required|string|min:8|confirmed',
        ]);
        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }
        $user = auth()->user();
        if (!Hash::check($request->current_password, $user->password)) {
            return $this->errorResponse('Le mot de passe actuel est incorrect.', 400);
        }
        $user->update(['password' => Hash::make($request->new_password)]);
        return $this->successResponse(null, 'Mot de passe modifié avec succès.');
    }

    protected function respondWithToken(string $token): JsonResponse
    {
        $user = auth()->user();
        return $this->successResponse([
            'access_token' => $token,
            'token_type'   => 'bearer',
            'expires_in'   => config('jwt.ttl') * 60,
            'user' => ['id' => $user->id, 'name' => $user->name, 'email' => $user->email, 'roles' => $user->getRoleNames()],
        ], 'Connexion réussie.');
    }
}

11.3 REGLE METIER - MODIFICATION BLOQUEE APRES 1 MINUTE

  $minutes = $infraction->created_at->diffInMinutes(now());

      );
  }

12.1 AUTHENTIFICATION JWT
---------------------------
Header requis pour toutes les routes protegees :
  Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...

Structure du payload JWT decode :
  {
    "iss": "http://localhost:8000",
    "iat": 1716235000,
    "exp": 1716238600,
    "sub": "1",
    "name": "Admin Principal",
    "email": "admin@gescrim.sn",
    "roles": ["super_admin"]
  }

La signature garantit l integrite : si quelqu un modifie le payload,
la signature ne correspondra plus et le token sera rejete (401).

12.3 HACHAGE DES MOTS DE PASSE
---------------------------------
Algorithme : Bcrypt avec 12 rounds (BCRYPT_ROUNDS=12 dans .env)
Cast automatique dans User.php : 'password' => 'hashed'
Assignation : $user->password = 'motdepasse'; -> hache automatiquement
Verification : Hash::check($motDePasse, $user->password)
              -> retourne true si correct, false sinon

12 rounds = environ 250ms par verification, ce qui est lent pour un
attaquant mais transparent pour un utilisateur legitime.

12.4 VALIDATION DES DONNEES
------------------------------
Toutes les donnees entrantes sont validees avec Validator::make()
avant tout traitement en base de donnees.
Regles frequentes :
  'email'           => 'required|email|unique:users'
  'service_id'      => 'required|exists:services,id'
  'date'            => 'required|date'
  'issue'           => 'required|in:Constatee,Deferee'
  'latitude'        => 'nullable|numeric|between:-90,90'
  'longitude'       => 'nullable|numeric|between:-180,180'
  'sync_status'     => 'nullable|in:pending,synced'
  'annee'           => 'required|integer|min:2000|max:2027'
  'new_password'    => 'required|string|min:8|confirmed'

La regle 'exists:table,colonne' protege contre les donnees orphelines :
elle verifie que l ID pointe vers un enregistrement existant en base.

13.1 MIDDLEWARE AUTH:API
--------------------------
Fourni par le package php-open-source-saver/jwt-auth.
Verifie automatiquement a chaque requete :
  1. Presence du header : Authorization: Bearer <token>
  2. Validite de la signature JWT (avec JWT_SECRET du .env)
  3. Non-expiration du token (claim exp)
  4. Existence de l utilisateur associe en base de donnees

Si la verification echoue, le middleware retourne une erreur JSON
interceptee dans bootstrap/app.php.

13.2 CONFIGURATION DANS LES ROUTES
--------------------------------------
Usage standard dans routes/api.php :
  Route::middleware('auth:api')->group(function () {
      // Toutes les routes ici requierent un token JWT valide
      Route::apiResource('infractions', InfractionController::class);
  });

Usage dans un constructeur de controleur :
  public function __construct() {
      // Toutes les methodes protegees sauf login
      $this->middleware('auth:api')->except(['login']);
  }

13.3 GESTION DES EXCEPTIONS JWT (bootstrap/app.php)
------------------------------------------------------
Chaque erreur JWT est transformee en reponse JSON :

  TokenExpiredException  -> HTTP 401 "Le token a expire."
  TokenInvalidException  -> HTTP 401 "Token invalide."
  JWTException           -> HTTP 401 "Token absent."
  NotFoundHttpException  -> HTTP 404 "Ressource introuvable."
  AccessDeniedHttpException -> HTTP 403 "Acces refuse."

Condition de declenchement : if ($request->is('api/*'))
Cela garantit que seules les routes API retournent du JSON.
Les routes web continuent de retourner des pages HTML.

13.4 CODES HTTP DE REFERENCE
-------------------------------
  200 : Succes - GET, PUT, DELETE reussis
  201 : Creation reussie - POST
  400 : Mauvaise requete - ex: mot de passe incorrect
  401 : Non authentifie - token absent, invalide ou expire
  403 : Acces refuse - pas les droits, delai depasse
  404 : Ressource introuvable
  422 : Erreur de validation des donnees entrantes
  500 : Erreur interne du serveur

14.5 TRANSACTIONS SQL POUR LA COHERENCE DES DONNEES
------------------------------------------------------
Les operations critiques (sync/batch, import/json) utilisent des
transactions pour garantir la coherence des donnees :

  DB::beginTransaction();
  try {
      foreach ($elements as $data) {
          Model::create($data);
      }
      DB::commit();
      return $this->successResponse($results, 'Operation reussie.');
  } catch (\Exception $e) {
      DB::rollBack(); // Annulation complete si une erreur survient
      return $this->errorResponse('Erreur: ' . $e->getMessage(), 500);
  }

Principe : tout passe ou rien. Si un seul element echoue, tous les
autres sont annules (rollback). Cela evite des donnees partiellement
inserees en base.

Actions enregistrees automatiquement :

Informations stockees pour chaque action :

16.2 FICHIER : tests/Feature/RegionTest.php
----------------------------------------------
Test fonctionnel qui verifie le CRUD des regions.
Utilise RefreshDatabase pour reinitialiser la BDD apres chaque test.

<?php
namespace Tests\Feature;
use App\Models\User;
use App\Models\Region;
use Illuminate\Foundation\Testing\RefreshDatabase;
use Tests\TestCase;

class RegionTest extends TestCase
{
    use RefreshDatabase;

    protected User $admin;
    protected string $token;

    protected function setUp(): void
    {
        parent::setUp();
        // Creer les roles et l utilisateur de test
        $this->seed(\Database\Seeders\RolePermissionSeeder::class);
        $this->admin = User::factory()->create(['is_active' => true]);
        $this->admin->assignRole('super_admin');
        // Obtenir un token JWT pour cet utilisateur
        $this->token = auth('api')->login($this->admin);
    }

    public function test_liste_regions_retourne_200(): void
    {
        Region::create(['nom' => 'Dakar', 'code' => 'DK']);
        $response = $this->withToken($this->token)
                         ->getJson('/api/regions');
        $response->assertStatus(200)
                 ->assertJsonStructure([
                     'success', 'message', 'data', 'meta'
                 ]);
    }

    public function test_creer_region_reussit(): void
    {
        $response = $this->withToken($this->token)
                         ->postJson('/api/regions', [
                             'nom'  => 'Ziguinchor',
                             'code' => 'ZG',
                         ]);
        $response->assertStatus(201)
                 ->assertJson(['success' => true]);
        $this->assertDatabaseHas('regions', ['nom' => 'Ziguinchor']);
    }

    public function test_creer_region_sans_token_retourne_401(): void
    {
        $response = $this->postJson('/api/regions', [
            'nom' => 'Thies', 'code' => 'TH',
        ]);
        $response->assertStatus(401);
    }

    public function test_supprimer_region_reussit(): void
    {
        $region = Region::create(['nom' => 'Kaolack', 'code' => 'KL']);
        $response = $this->withToken($this->token)
                         ->deleteJson("/api/regions/{$region->id}");
        $response->assertStatus(200)
                 ->assertJson(['success' => true]);
        $this->assertDatabaseMissing('regions', ['id' => $region->id]);
    }
}

SECTION 20 — RESOLUTION DES ERREURS FREQUENTES
================================================================================

ERREUR : "Class 'PHPOpenSourceSaver\JWTAuth\...' not found"
CAUSE   : Le package JWT n est pas installe ou l autoload n est pas regenere.
SOLUTION:
  composer require php-open-source-saver/jwt-auth
  composer dump-autoload
  php artisan vendor:publish --provider="PHPOpenSourceSaver\JWTAuth\Providers\LaravelServiceProvider"
  php artisan jwt:secret

----------------------------------------------------------------------
ERREUR : "SQLSTATE[08006] Connection refused" (PostgreSQL)
CAUSE   : PostgreSQL n est pas demarre ou les identifiants sont incorrects.
SOLUTION:
  sudo systemctl start postgresql
  sudo systemctl status postgresql
  # Verifier dans .env : DB_HOST=127.0.0.1, DB_PORT=5432
  # Tester la connexion :
  psql -U postgres -d teranga_gescrim -h 127.0.0.1

----------------------------------------------------------------------
ERREUR : "php_network_getaddresses: getaddrinfo for pgsql failed"
CAUSE   : DB_HOST est mal configure (ex: localhost au lieu de 127.0.0.1).
SOLUTION: Dans .env, remplacer DB_HOST=localhost par DB_HOST=127.0.0.1

----------------------------------------------------------------------
ERREUR : "Token has expired" apres 1 heure
CAUSE   : Par defaut, les tokens JWT expirent apres 60 minutes.
SOLUTION:
  # Option 1 : Appeler /api/auth/refresh avant expiration
  POST /api/auth/refresh
  Authorization: Bearer <token_bientot_expire>

  # Option 2 : Allonger la duree de vie dans config/jwt.php
  'ttl' => env('JWT_TTL', 1440),  # 1440 minutes = 24 heures
  # Et dans .env :
  JWT_TTL=1440

----------------------------------------------------------------------
ERREUR : "CORS policy: No 'Access-Control-Allow-Origin' header"
CAUSE   : Le navigateur bloque les requetes cross-origin.
SOLUTION: Verifier config/cors.php :
  'paths' => ['api/*'],
  'allowed_origins' => ['*'],  // ou l URL specifique du frontend
  'allowed_headers' => ['*'],
  Puis : php artisan config:clear

----------------------------------------------------------------------
ERREUR : "419 CSRF token mismatch"
CAUSE   : L application tente d utiliser la protection CSRF (mode web).
SOLUTION: Utiliser uniquement les routes prefixees /api/ qui n ont
          pas de protection CSRF. Ne pas appeler des routes web depuis
          un client mobile ou Angular.

----------------------------------------------------------------------
ERREUR : "The mix manifest does not exist"
CAUSE   : Vite ou Mix n a pas ete execute (inutile pour une API pure).
SOLUTION: Ce projet etant une API, les assets JS/CSS ne sont pas
          utilises. Ignorer cette erreur ou supprimer vite.config.js.

----------------------------------------------------------------------
ERREUR : "Spatie\Permission\Exceptions\PermissionAlreadyExists"
CAUSE   : Le seeder de permissions est execute deux fois.
SOLUTION:
  # Option 1 : Effacer le cache des permissions
  php artisan cache:clear

  # Option 2 : Vider les tables avant de re-seeder
  php artisan migrate:fresh --seed

----------------------------------------------------------------------
ERREUR : "Class 'OpenApi\Annotations\...' not found" (Swagger)
CAUSE   : doctrine/annotations n est pas installe.
SOLUTION:
  composer require doctrine/annotations
  php artisan l5-swagger:generate

----------------------------------------------------------------------
ERREUR : "Storage link does not exist"
CAUSE   : Le lien symbolique public/storage n existe pas.
SOLUTION: php artisan storage:link

21.1 FLUX D UNE REQUETE HTTP DE A A Z
---------------------------------------
Voici ce qui se passe exactement quand un agent mobile envoie
une infraction depuis l application Flutter :

1. L AGENT MOBILE (Flutter)
   L agent remplit le formulaire d infraction. L app Flutter
   construit une requete HTTP POST avec le token JWT dans le header.

2. TRANSMISSION RESEAU
   POST http://192.168.1.100:8000/api/infractions
   Headers : Authorization: Bearer eyJ0eXAiOiJKV1Qi...
   Body    : { "type_infraction_id": 5, "date": "2025-05-19", ... }

3. POINT D ENTREE LARAVEL (public/index.php)
   Ce fichier est le seul point d entree de toute l application.
   Il initialise le framework et passe la requete au routeur.

4. BOOTSTRAP (bootstrap/app.php)
   Configure l application : routes, middlewares, gestion des erreurs.

5. ROUTEUR (routes/api.php)
   Identifie que POST /api/infractions correspond a :
   Route::apiResource('infractions', InfractionController::class)
   -> Methode InfractionController::store()

6. MIDDLEWARE AUTH:API
   Intercepte la requete avant d appeler le controleur.
   Verifie le token JWT : signature, expiration, utilisateur.
   Si valide : passe la requete au controleur.
   Si invalide : retourne 401 immediatement.

7. CONTROLEUR : InfractionController::store()
   a) Valide les donnees avec Validator::make()
   b) Ajoute user_id = ID de l utilisateur authentifie
   c) Cree l infraction : Infraction::create($data)

8. MODELE ELOQUENT (Infraction)
   Execute la requete SQL INSERT via l ORM :
   INSERT INTO infractions (type_infraction_id, date, ...) VALUES (5, '2025-05-19', ...)

9. TRAIT AUDITABLE (automatique)
   L evenement "created" se declenche automatiquement.
   Le trait Auditable insere un log dans audit_logs :
   INSERT INTO audit_logs (user_id, action, model_type, ...) VALUES (3, 'create', 'Infraction', ...)

10. REPONSE JSON
    Le controleur retourne :
    HTTP 201
    { "success": true, "message": "Infraction enregistree.", "data": {...} }

11. RECEPTION PAR L APP MOBILE
    Flutter recoit la reponse 201 et affiche un message de succes.
    L infraction est marquee comme "synced" localement.

21.2 FONCTIONNEMENT DU MODE OFFLINE
--------------------------------------
Quand un agent est sans connexion internet :

1. L agent saisit une infraction dans l app Flutter.
2. Flutter stocke l infraction localement (SQLite ou SharedPreferences)
   avec sync_status = "pending".
3. Quand la connexion est retablie, l app envoie tout en une fois :
   POST /api/sync/batch
   Body : { "infractions": [...liste des infractions pending...] }
4. SyncController::batch() traite chaque infraction dans une transaction.
5. Les infractions sont inserees en base avec sync_status = "synced".
6. L app mobile met a jour son stockage local : status = "synced".

<?php

namespace App\Http\Controllers\Api;

use App\Models\Accident;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Validator;
use OpenApi\Annotations as OA;

/**
 * @OA\Tag(name="Accidents")
 */
class AccidentController extends ApiController
{
    /**
     * @OA\Get(
     *     path="/api/accidents",
     *     tags={"Accidents"},
     *     summary="Liste des accidents de la circulation",
     *     security={{"bearerAuth":{}}},
     *     @OA\Parameter(name="type", in="query", required=false, @OA\Schema(type="string", enum={"mat\u00e9riel","corporel","mortel"})),
     *     @OA\Parameter(name="service_id", in="query", required=false, @OA\Schema(type="integer")),
     *     @OA\Parameter(name="commune_id", in="query", required=false, @OA\Schema(type="integer")),
     *     @OA\Parameter(name="date_from", in="query", required=false, @OA\Schema(type="string", format="date")),
     *     @OA\Parameter(name="date_to", in="query", required=false, @OA\Schema(type="string", format="date")),
     *     @OA\Response(response=200, description="Liste paginee", @OA\JsonContent(ref="#/components/schemas/SuccessResponse")),
     *     @OA\Response(response=401, description="Non authentifié")
     * )
     */
    public function index(Request $request): JsonResponse
    {
        $query = Accident::with(['service', 'commune', 'user']);

        if ($request->has('type')) {
            $query->byType($request->type);
        }
        if ($request->has('service_id')) {
            $query->byService($request->service_id);
        }
        if ($request->has('commune_id')) {
            $query->byCommune($request->commune_id);
        }
        if ($request->has('date_from') && $request->has('date_to')) {
            $query->byDateRange($request->date_from, $request->date_to);
        }

        $accidents = $query->withCount('victimes')
            ->orderByDesc('date')
            ->paginate($request->get('per_page', 15));

        return $this->paginatedResponse($accidents);
    }

    /**
     * @OA\Get(
     *     path="/api/accidents/{id}",
     *     tags={"Accidents"},
     *     summary="Détail d'un accident",
     *     security={{"bearerAuth":{}}},
     *     @OA\Parameter(name="id", in="path", required=true, @OA\Schema(type="integer")),
     *     @OA\Response(response=200, description="Détail"),
     *     @OA\Response(response=404, description="Introuvable")
     * )
     */
    public function show(Accident $accident): JsonResponse
    {
        $accident->load(['service', 'commune.departement.region', 'user', 'victimes', 'media']);
        return $this->successResponse($accident);
    }

    /**
     * @OA\Post(
     *     path="/api/accidents",
     *     tags={"Accidents"},
     *     summary="Enregistrer un accident",
     *     security={{"bearerAuth":{}}},
     *     @OA\RequestBody(
     *         required=true,
     *         @OA\JsonContent(
     *             required={"type","date","lieu","commune_id","service_id"},
     *             @OA\Property(property="type", type="string", enum={"matériel","corporel","mortel"}, example="corporel"),
     *             @OA\Property(property="date", type="string", format="date", example="2025-05-12"),
     *             @OA\Property(property="lieu", type="string", example="Autoroute à péage, km 15"),
     *             @OA\Property(property="commune_id", type="integer", example=1),
     *             @OA\Property(property="service_id", type="integer", example=1),
     *             @OA\Property(property="moyen", type="string", example="Véhicule"),
     *             @OA\Property(property="cause_probable", type="string", example="Excès de vitesse"),
     *             @OA\Property(property="latitude", type="number", example=14.7228),
     *             @OA\Property(property="longitude", type="number", example=-17.4567)
     *         )
     *     ),
     *     @OA\Response(response=201, description="Créé", @OA\JsonContent(ref="#/components/schemas/SuccessResponse")),
     *     @OA\Response(response=422, description="Validation", @OA\JsonContent(ref="#/components/schemas/ErrorResponse"))
     * )
     */
    public function store(Request $request): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'type' => 'required|in:matériel,corporel,mortel',
            'date' => 'required|date',
            'lieu' => 'required|string|max:500',
            'commune_id' => 'required|exists:communes,id',
            'service_id' => 'required|exists:services,id',
            'moyen' => 'nullable|string|max:255',
            'cause_probable' => 'nullable|string',
            'latitude' => 'nullable|numeric|between:-90,90',
            'longitude' => 'nullable|numeric|between:-180,180',
            'description' => 'nullable|string',
            'sync_status' => 'nullable|in:pending,synced',
        ], [
            'type.required' => 'Le type d\'accident est obligatoire.',
            'type.in' => 'Le type doit être matériel, corporel ou mortel.',
            'date.required' => 'La date est obligatoire.',
            'lieu.required' => 'Le lieu est obligatoire.',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $data = $request->all();
        $data['user_id'] = auth()->id();

        $accident = Accident::create($data);

        return $this->successResponse(
            $accident->load(['service', 'commune']),
            'Accident enregistré avec succès.',
            201
        );
    }

    /**
     * @OA\Put(
     *     path="/api/accidents/{id}",
     *     tags={"Accidents"},
     *     summary="Modifier un accident",
     *     security={{"bearerAuth":{}}},
     *     @OA\Parameter(name="id", in="path", required=true, @OA\Schema(type="integer")),
     *     @OA\Response(response=200, description="Mis à jour")
     * )
     */
    public function update(Request $request, Accident $accident): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'type' => 'sometimes|in:matériel,corporel,mortel',
            'date' => 'sometimes|date',
            'lieu' => 'sometimes|string|max:500',
            'commune_id' => 'sometimes|exists:communes,id',
            'service_id' => 'sometimes|exists:services,id',
            'moyen' => 'nullable|string|max:255',
            'cause_probable' => 'nullable|string',
            'latitude' => 'nullable|numeric|between:-90,90',
            'longitude' => 'nullable|numeric|between:-180,180',
            'description' => 'nullable|string',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $accident->update($request->all());

        return $this->successResponse(
            $accident->load(['service', 'commune']),
            'Accident mis à jour avec succès.'
        );
    }

    /**
     * @OA\Delete(
     *     path="/api/accidents/{id}",
     *     tags={"Accidents"},
     *     summary="Supprimer un accident",
     *     security={{"bearerAuth":{}}},
     *     @OA\Parameter(name="id", in="path", required=true, @OA\Schema(type="integer")),
     *     @OA\Response(response=200, description="Supprimé")
     * )
     */
    public function destroy(Accident $accident): JsonResponse
    {
        $accident->delete();
        return $this->successResponse(null, 'Accident supprimé avec succès.');
    }
}

<?php

namespace App\Http\Controllers\Api;

use App\Models\Commune;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\Validator;

/**
 * Contrôleur CRUD pour les communes.
 */
class CommuneController extends ApiController
{
    public function index(Request $request): JsonResponse
    {
        $query = Commune::with('departement.region');

        if ($request->has('search')) {
            $query->search($request->search);
        }
        if ($request->has('departement_id')) {
            $query->byDepartement($request->departement_id);
        }

        $communes = $query->withCount('services')
            ->orderBy('nom')
            ->paginate($request->get('per_page', 15));

        return $this->paginatedResponse($communes);
    }

    public function all(Request $request): JsonResponse
    {
        $query = Commune::with('departement')->orderBy('nom');
        if ($request->has('departement_id')) {
            $query->byDepartement($request->departement_id);
        }
        return $this->successResponse($query->get());
    }

    public function show(Commune $commune): JsonResponse
    {
        $commune->load(['departement.region', 'services']);
        return $this->successResponse($commune);
    }

    public function store(Request $request): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'nom' => 'required|string|max:255',
            'code' => 'nullable|string|max:10|unique:communes,code',
            'departement_id' => 'required|exists:departements,id',
        ], [
            'nom.required' => 'Le nom de la commune est obligatoire.',
            'departement_id.exists' => 'Le département sélectionné n\'existe pas.',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $commune = Commune::create($request->only('nom', 'code', 'departement_id'));

        return $this->successResponse($commune->load('departement'), 'Commune créée avec succès.', 201);
    }

    public function update(Request $request, Commune $commune): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'nom' => 'sometimes|string|max:255',
            'code' => 'nullable|string|max:10|unique:communes,code,' . $commune->id,
            'departement_id' => 'sometimes|exists:departements,id',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $commune->update($request->only('nom', 'code', 'departement_id'));

        return $this->successResponse($commune->load('departement'), 'Commune mise à jour avec succès.');
    }

    public function destroy(Commune $commune): JsonResponse
    {
        if ($commune->services()->exists()) {

<?php

namespace App\Http\Controllers\Api;

use App\Models\Infraction;
use App\Models\Accident;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Validator;
use OpenApi\Annotations as OA;

/**
 * Contrôleur pour la synchronisation offline.
 * Gère l'envoi par lot des données saisies hors ligne.
 */
class SyncController extends ApiController
{
    /**
     * @OA\Post(
     *     path="/api/sync/batch",
     *     tags={"Synchronisation"},
     *     summary="Synchroniser les données offline par lot",
     *     description="Envoie en une seule requête toutes les infractions et accidents saisis en mode hors-ligne.",
     *     security={{"bearerAuth":{}}},
     *     @OA\RequestBody(
     *         required=true,
     *         @OA\JsonContent(
     *             @OA\Property(property="infractions", type="array", @OA\Items(ref="#/components/schemas/Infraction")),
     *             @OA\Property(property="accidents", type="array", @OA\Items(ref="#/components/schemas/Accident"))
     *         )
     *     ),
     *     @OA\Response(response=200, description="Synchronisation réussie",
     *         @OA\JsonContent(@OA\Property(property="infractions_synced", type="integer"),
     *                         @OA\Property(property="accidents_synced", type="integer"))
     *     ),
     *     @OA\Response(response=422, description="Erreur de validation", @OA\JsonContent(ref="#/components/schemas/ErrorResponse"))
     * )
     */
    public function batch(Request $request): JsonResponse
    {
        $validator = Validator::make($request->all(), [
            'infractions' => 'nullable|array',
            'infractions.*.type_infraction_id' => 'required|exists:type_infractions,id',
            'infractions.*.service_id' => 'required|exists:services,id',
            'infractions.*.date' => 'required|date',
            'infractions.*.lieu' => 'required|string',
            'infractions.*.commune_id' => 'required|exists:communes,id',
            'infractions.*.issue' => 'required|in:Constatée,Déférée',
            'accidents' => 'nullable|array',
            'accidents.*.type' => 'required|in:matériel,corporel,mortel',
            'accidents.*.date' => 'required|date',
            'accidents.*.lieu' => 'required|string',
            'accidents.*.commune_id' => 'required|exists:communes,id',
            'accidents.*.service_id' => 'required|exists:services,id',
        ]);

        if ($validator->fails()) {
            return $this->errorResponse('Erreur de validation', 422, $validator->errors());
        }

        $results = ['infractions_synced' => 0, 'accidents_synced' => 0, 'errors' => []];

        DB::beginTransaction();
        try {
            // Synchroniser les infractions
            if ($request->has('infractions')) {
                foreach ($request->infractions as $index => $data) {
                    $data['user_id'] = auth()->id();
                    $data['sync_status'] = 'synced';
                    $data['annee'] = date('Y', strtotime($data['date']));
                    Infraction::create($data);
                    $results['infractions_synced']++;
                }
            }

            // Synchroniser les accidents
            if ($request->has('accidents')) {
                foreach ($request->accidents as $data) {
                    $data['user_id'] = auth()->id();
                    $data['sync_status'] = 'synced';
                    Accident::create($data);
                    $results['accidents_synced']++;
                }
            }

            DB::commit();
            return $this->successResponse($results, 'Synchronisation réussie.');
        } catch (\Exception $e) {
            DB::rollBack();
            return $this->errorResponse('Erreur de synchronisation: ' . $e->getMessage(), 500);
        }
    }

    /**
     * @OA\Get(
     *     path="/api/sync/status",
     *     tags={"Synchronisation"},
     *     summary="État de synchronisation",
     *     description="Retourne le nombre d'éléments en attente et synchonisés.",
     *     security={{"bearerAuth":{}}},
     *     @OA\Response(response=200, description="Statut de sync")
     * )
     */
    public function status(): JsonResponse
    {
        $data = [
            'pending_infractions' => Infraction::pending()->count(),
            'pending_accidents' => Accident::pending()->count(),
            'synced_infractions' => Infraction::synced()->count(),
            'synced_accidents' => Accident::synced()->count(),
        ];
        return $this->successResponse($data);
    }
}
